## Goal

Ensure the caching and data refresh strategy is solid for MVP. For MVP, we rely on KV TTL expiration (24hr) + the cache-miss fallback already built in `functions/api/data.js`. No separate cron worker is needed yet. This phase verifies and hardens what Phase 2 built, and stubs out the future cron worker path.

## Context — What Phase 2 Already Did

The `functions/api/data.js` already implements:
- KV cache read with `env.CACHE.get("all_data", "json")`
- `expirationTtl: 86400` on KV writes (24hr auto-expiry)
- Tranco history accumulation: reads `tranco_history` from KV, merges new ranks, writes back (no TTL on history key)
- Cache-miss fallback: fetches live from Radar + Tranco, saves to KV, returns

## Steps

### 1. Verify `expirationTtl: 86400` is correctly set

**File:** `functions/api/data.js:164`

Confirm the KV write uses `expirationTtl: 86400` (24 hours in seconds). This is already implemented:
```js
await env.CACHE.put("all_data", JSON.stringify(data), { expirationTtl: 86400 });
```
No changes needed — just verification.

### 2. Verify Tranco historical accumulation on cache-miss

**File:** `functions/api/data.js:73-101`

The `fetchTrancoData()` function already:
1. Reads existing `tranco_history` from KV
2. Fetches today's ranks from Tranco API
3. Merges via `mergeHistory()` (dedup by date, sorted chronologically)
4. Writes updated history back to KV (no TTL — permanent accumulation store)

This means every time the `all_data` cache expires (24hr) and a visitor triggers a cache-miss, fresh Tranco data is fetched and appended to history. Over days/weeks, this builds up the Tranco time series.

No changes needed — just verification.

### 3. Add a `_refresh` query param for manual cache busting

**File:** `functions/api/data.js`

Add support for `?refresh=true` query param to bypass the KV cache and force a live fetch. Useful for:
- Testing after deploy
- Manual data refresh without waiting for TTL expiry
- Debugging data issues

Modify `onRequestGet` to check the URL for `?refresh=true` and skip the cache read when present.
