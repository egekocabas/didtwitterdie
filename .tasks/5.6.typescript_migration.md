## Problem

The entire codebase is JavaScript with no type safety. API response shapes, component props, and utility functions are all untyped. This means:
- No compile-time validation of API response shapes
- No autocomplete or type checking for component props
- Easy to pass wrong data shapes between components without knowing

## Scope

Migrate all `.js`/`.jsx` files to TypeScript with strict mode. Define shared types for the API response, component props, and Cloudflare bindings.

---

## Prerequisites (before running the task)

None — all dependencies can be installed via npm.

## Steps

### Step 1: Install dependencies

```bash
npm install -D typescript @cloudflare/workers-types typescript-eslint
```

- `typescript` — the compiler
- `@cloudflare/workers-types` — types for `KVNamespace`, Pages Functions context
- `typescript-eslint` — ESLint parser + plugin for `.ts`/`.tsx` files

Note: `@types/react` and `@types/react-dom` are already installed.

### Step 2: Replace `jsconfig.json` with `tsconfig.json`

Delete `jsconfig.json`, create `tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "types": ["@cloudflare/workers-types"],
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src", "functions"]
}
```

Key flags:
- `strict: true` — enables all strict checks
- `noUncheckedIndexedAccess: true` — forces null checks on array/object access
- `resolveJsonModule: true` — allows importing `mockData.json`
- `types: ["@cloudflare/workers-types"]` — provides `KVNamespace`, `EventContext`, etc.

### Step 3: Create shared type definitions

Create `src/types.ts`:

```typescript
// API data shapes
export interface RankEntry {
  date: string;
  rank: number;
}

export interface RadarDomainInfo {
  bucket: string | null;
}

export interface TrancoData {
  twitter: RankEntry[];
  x: RankEntry[];
}

export interface RadarData {
  twitter: RadarDomainInfo;
  x: RadarDomainInfo;
}

export interface ApiResponse {
  radar: RadarData | null;
  trends: null;
  tranco: TrancoData | null;
  updated_at: number;
  errors?: string[];
}

// UI types
export type TimeRange = "1M" | "6M" | "1Y" | "3Y" | "ALL";
```

Create `functions/types.ts` (Cloudflare-specific):

```typescript
export interface Env {
  CLOUDFLARE_RADAR_TOKEN: string;
  CACHE: KVNamespace;
}
```

### Step 5: Add types to each file

**Priority order** (types flow top-down):

1. **`src/types.ts`** — shared types (created in Step 3)
2. **`functions/types.ts`** + **`functions/api/data.ts`** — backend types + typed API handler
3. **`src/hooks/useData.ts`** — typed hook return `{ data: ApiResponse | null; loading: boolean; error: string | null }`
4. **`src/utils/filterByRange.ts`** — typed with `RankEntry[]` and `TimeRange`
5. **Components** — add typed props interfaces:

| Component | Props Interface |
|-----------|----------------|
| `Layout` | `{ children: React.ReactNode }` |
| `ChartWrapper` | `{ title?: string; description?: string; children: React.ReactNode }` |
| `TimeRangeSelector` | `{ value: TimeRange; onChange: (range: TimeRange) => void }` |
| `Footer` | `{ updatedAt?: number \| null }` |
| `HeroSection` | `{ radar: RadarData \| null; tranco: TrancoData \| null }` |
| `RankingChart` | `{ data: TrancoData \| null }` |
| `VerdictSection` | `{ data: ApiResponse }` |
| `TrendsChart` | (no props) |
| `RebrandAnnotation` | (no props) |
| `LoadingSkeleton` | (no props) |

Internal types within components:
- `RankingChart`: `MergedRankEntry = { date: string; twitter?: number; x?: number }`
- `VerdictSection`: `Verdict = { twitterRank: number; xRank: number; twitterWins: boolean; alivePercent: number; twitterBucket: string | null; xBucket: string | null }`
- `App`: `ErrorStateProps = { message: string }`

### Step 6: Update ESLint config

Update `eslint.config.js`:
- Change file glob from `**/*.{js,jsx}` to `**/*.{ts,tsx}`
- Add `typescript-eslint` parser and plugin
- Keep existing React plugins

### Step 7: Update vite.config

Rename `vite.config.js` to `vite.config.ts` (optional but consistent). Vite supports TypeScript natively — no extra plugins needed.

### Step 8: Verify

```bash
npx tsc --noEmit          # type check — should pass with 0 errors
npm run lint              # eslint — should pass
npm run build             # vite build — should produce dist/
```
