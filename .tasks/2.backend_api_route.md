## Goal

Create the `functions/api/data.js` Pages Function that serves as the single GET `/api/data` endpoint. It should read from KV cache first, fall back to live fetching from Cloudflare Radar + Tranco APIs on cache miss, cache the result in KV, and return the full JSON payload. Also create a mock data file for frontend development.

## Steps

### 1. Create mock data file (`src/utils/mockData.json`)

Create a JSON file matching the exact API response shape documented in the README (lines 483-517). Include:
- `radar.twitter[]` — array of `{ date, rank }` objects from Sept 2022 to present (~weekly)
- `radar.x[]` — array of `{ date, rank }` starting from July 2023 rebrand
- `radar.regional` — object with ~10 country keys, each having `twitter_rank` and `x_rank`
- `trends: null` — deferred to Phase 9
- `tranco.twitter[]` and `tranco.x[]` — arrays of `{ date, rank }`
- `updated_at` — timestamp

This mock data allows frontend development to proceed independently of real API keys.

### 2. Create `functions/api/data.js` — main GET endpoint

File: `functions/api/data.js`

Export `onRequestGet({ env })` as the Cloudflare Pages Function handler.

```
export async function onRequestGet({ env }) {
  // 1. Try KV cache first
  // 2. Cache miss → fetch live → save to KV → return
  // 3. Return JSON with proper headers
}
```

### 3. Implement KV cache read logic

Inside `onRequestGet`:
- Read `env.CACHE.get("all_data", "json")`
- If cache hit, return immediately with:
  - `Content-Type: application/json`
  - `Access-Control-Allow-Origin: *`
  - `Cache-Control: public, max-age=3600`

### 4. Implement `fetchRadarData(env)`

Call the Cloudflare Radar API using `env.CLOUDFLARE_RADAR_TOKEN` from `.dev.vars`.

**Three API calls needed:**

a) **Time series** (historical rank for both domains):
```
GET https://api.cloudflare.com/client/v4/radar/ranking/timeseries_groups
  ?domains=twitter.com,x.com
  &dateRange=52w (or use dateStart/dateEnd for full history from 2022-09-26)
Headers: Authorization: Bearer {token}
```

b) **Current rank** for each domain:
```
GET https://api.cloudflare.com/client/v4/radar/ranking/domain/twitter.com
GET https://api.cloudflare.com/client/v4/radar/ranking/domain/x.com
```

c) **Regional data** — fetch per-country rank for ~10 key countries using the `location` parameter:
- Countries: US, GB, JP, BR, DE, FR, IN, CA, AU, TR (good geographic spread)
- Can batch these calls with `Promise.all`

**Return shape:**
```json
{
  "twitter": [{ "date": "2022-09-26", "rank": 5 }, ...],
  "x": [{ "date": "2023-07-30", "rank": 45 }, ...],
  "regional": {
    "US": { "twitter_rank": 5, "x_rank": 10 },
    ...
  }
}
```

### 5. Implement `fetchTrancoData(env)`

Call the Tranco API (no auth required).

**API calls:**
```
GET https://tranco-list.eu/api/ranks/domain/twitter.com
GET https://tranco-list.eu/api/ranks/domain/x.com
```

For historical accumulation:
1. Read existing Tranco history from KV (`env.CACHE.get("tranco_history", "json")`)
2. Fetch today's rank for both domains
3. Append today's entry if not already present (dedup by date)
4. Write updated history back to KV

**Return shape:**
```json
{
  "twitter": [{ "date": "2024-01-01", "rank": 4 }, ...],
  "x": [{ "date": "2024-01-01", "rank": 18 }, ...]
}
```

### 6. Implement cache-miss fallback

When KV cache is empty:
1. Call `fetchRadarData(env)` and `fetchTrancoData(env)` in parallel via `Promise.all`
2. Assemble the full response object:
   ```json
   {
     "radar": { ... },
     "trends": null,
     "tranco": { ... },
     "updated_at": Date.now()
   }
   ```
3. Write to KV with `expirationTtl: 86400` (24 hours)
4. Return the JSON response

### 7. Add error handling

- Wrap each data source fetch in try/catch
- If Radar fails: return `radar: null` with an error flag
- If Tranco fails: return `tranco: null` with an error flag
- Never let one source failure break the entire endpoint
- Return partial data with `errors` array indicating which sources failed:
  ```json
  {
    "radar": null,
    "tranco": { ... },
    "trends": null,
    "errors": ["radar: fetch failed — 401 Unauthorized"],
    "updated_at": Date.now()
  }
  ```

### 8. Set response headers

All responses should include:
- `Content-Type: application/json`
- `Access-Control-Allow-Origin: *` (needed for local dev CORS)
- `Cache-Control: public, max-age=3600` (browser caches for 1 hour)

## Acceptance Criteria

- `GET /api/data` returns valid JSON matching the documented response shape
- KV cache is checked first; cache hit returns in ~1ms with no external API calls
- On cache miss, Radar and Tranco are fetched in parallel
- Response is saved to KV with 24hr TTL after live fetch
- If one data source fails, the endpoint still returns partial data with error flags
- `Cache-Control` header is set for browser-side caching
- CORS headers are present on all responses
- Mock data file exists and matches the API response shape exactly
- `npm run dev` serves the endpoint at `http://localhost:8788/api/data`
