## Problem

During Phase 5.2 local testing, we discovered that the Cloudflare Radar ranking API **cannot** provide exact rank time series for twitter.com or x.com. The `timeseries_groups` endpoint only returns data for the top ~110 domains, which are dominated by CDN/infrastructure domains (google.com, googleapis.com, cloudflare.com, etc.). Neither twitter.com nor x.com appears in this list.

### Full Radar API Audit (verified with live API calls)

**What Radar CAN provide:**
- **Bucket data** via `GET /radar/ranking/domain/{domain}`: twitter.com = "top 200", x.com = "top 500" — a meaningful signal that twitter.com gets more DNS queries
- **Internet Services ranking** via `GET /radar/ranking/internet_services/top`: "X / Twitter" combined = rank #31 among internet services. Weekly time series available (max ~1 year). Shows the platform declining from #25 → #31 over the past year.
- **Bucket dataset confirmation** via `GET /radar/datasets/ranking_top_200`: twitter.com is confirmed in the top 200 list. x.com confirmed in top 500 but NOT top 200.

**What Radar CANNOT provide (all tested, all confirmed dead ends):**
- Exact rank numbers for twitter.com/x.com (rank is always `null`, only bucket returned)
- Time series for these specific domains (`timeseries_groups` only covers top ~110)
- Per-country breakdown (bucket, DNS locations, and Internet Services rank are all identical globally — tested US, JP, TR, BR)
- Per-domain DNS query volume (`/radar/dns/timeseries` only accepts TLD filter, not domain)
- Per-domain HTTP traffic (`/radar/http/timeseries` is aggregate only, no domain filter)

### Tranco API — Better Than Expected

**The Tranco API has a hidden gem: historical list downloads.** Beyond the 39-day rolling window from `/api/ranks/domain/{domain}`, you can download ANY historical Tranco list by date:

1. `GET /api/lists/date/YYYY-MM-DD` → returns list ID
2. `GET /download/{list_id}/1000` → CSV of top 1000 domains with exact ranks

### Current Frontend Issues

- **Two near-identical chart components** (DnsChart for Radar, RankingChart for Tranco) with duplicated `mergeByDate`/`formatDate` helpers
- **DnsChart has no real data source** (Radar can't provide time series)
- **RegionalChart** relied on per-country Radar data that's identical everywhere
- **Mock data** has fake rank numbers (twitter.com rank 5, x.com rank 48) that don't match reality

## Solution

1. **Radar** → demote to supplementary metric (bucket badge in Hero + Verdict)
2. **Tranco** → promote to primary and only time series data source
3. **Pre-seed Tranco history** → create a seed data file with quarterly historical data (2022–2026) from verified list downloads, so the chart is rich from day one instead of only showing 39 days
4. **Merge chart sections** → one chart (Tranco) instead of two identical chart components
5. **Remove RegionalChart** → defer to Phase 9 when Google Trends (which has regional data) is added
6. **Update verdict** → compute from Tranco data instead of Radar

**Page sections after this change: 4 instead of 6**
1. Hero (Tranco ranks + Radar bucket badge)
2. Domain popularity chart (Tranco time series — the only chart)
3. Google Trends placeholder (unchanged)
4. Verdict (computed from Tranco + Radar bucket context)

**New `fetchRadarData`:**
```js
async function fetchRadarData(env) {
  const token = env.CLOUDFLARE_RADAR_TOKEN;
  const headers = { Authorization: `Bearer ${token}` };

  const [twitterRes, xRes] = await Promise.all([
    fetch(`${RADAR_BASE}/domain/twitter.com`, { headers }),
    fetch(`${RADAR_BASE}/domain/x.com`, { headers }),
  ]);

  if (!twitterRes.ok || !xRes.ok) {
    throw new Error(`Radar domain fetch failed: twitter=${twitterRes.status} x=${xRes.status}`);
  }

  const [twitterJson, xJson] = await Promise.all([twitterRes.json(), xRes.json()]);

  return {
    twitter: { bucket: twitterJson?.result?.details_0?.bucket ?? null },
    x: { bucket: xJson?.result?.details_0?.bucket ?? null },
  };
}
```

```js
function computeVerdict(data) {
  const twitterRank = data?.tranco?.twitter?.at(-1)?.rank ?? null;
  const xRank = data?.tranco?.x?.at(-1)?.rank ?? null;
  if (twitterRank == null || xRank == null) return null;

  const twitterWins = twitterRank < xRank;
  const alivePercent = Math.round((1 - twitterRank / (twitterRank + xRank)) * 100);

  // Also include radar bucket as supplementary info
  const twitterBucket = data?.radar?.twitter?.bucket ?? null;
  const xBucket = data?.radar?.x?.bucket ?? null;

  return { twitterRank, xRank, twitterWins, alivePercent, twitterBucket, xBucket };
}
```
