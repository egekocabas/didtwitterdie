## Goal

Thoroughly test the entire application locally using Wrangler (Cloudflare's CLI) before deploying to Cloudflare Pages in Phase 6. Verify the frontend, backend API, KV caching, dark mode, mobile layout, and production build.

## Prerequisites

Before you begin, make sure you have:

1. **Cloudflare account** (free) — sign up at https://dash.cloudflare.com
2. **Cloudflare API token** for Radar — create one at https://dash.cloudflare.com/profile/api-tokens (using https://dash.cloudflare.com/82dfe8a2ceb0b4fb8a4e19c5e698e84d/api-tokens)
   - Use the "Custom token" template
   - Permissions needed: `Account > Cloudflare Radar > Read`
3. **Wrangler authenticated** — run `npx wrangler login` (opens a browser to authorize)

## Testing Steps

### Step 1: Set up `.dev.vars`

Copy the example file and fill in your real Cloudflare API token:

```bash
cp .dev.vars.example .dev.vars
```

Edit `.dev.vars`:
```
CLOUDFLARE_RADAR_TOKEN=your-real-token-here
```

This file is gitignored — it stays local only.

### Step 2: Start the dev server

There are two modes:

**Mode A — Frontend-only dev (fast hot reload, uses mock data):**
```bash
npm run dev
```
This runs Vite only at **http://localhost:5173**. The API endpoint `/api/data` won't exist, so the `useData` hook falls back to mock data automatically. Use this for frontend/UI work.

**Mode B — Full stack dev (Wrangler + KV + real API, no hot reload):**
```bash
npm run dev:full
```
This builds the Vite app first, then starts Wrangler Pages dev server at **http://localhost:8788**. This simulates the full Cloudflare environment:
- Pages Functions at `/api/*`
- Local KV store (persisted in `.wrangler/state/`)
- Reads secrets from `.dev.vars`

Use Mode B for testing the API, KV caching, and the full integration.

> **Note:** `wrangler pages dev -- vite` (the old proxy-command approach) is deprecated in Wrangler 4.78. The `command` positional is no longer supported. The correct approach is to build first, then serve `dist`.

### Step 3: Test the API endpoint

Open a new terminal and test the API:

```bash
# Test the API endpoint — should return real data from Cloudflare Radar + Tranco
curl -s http://localhost:8788/api/data | head -c 500

# Verify the response shape has all expected fields
curl -s http://localhost:8788/api/data | python3 -m json.tool | head -30
```

**What to check:**
- [ ] Response is valid JSON
- [ ] `radar.twitter` is an array of `{ date, rank }` objects
- [ ] `radar.x` is an array of `{ date, rank }` objects
- [ ] `radar.regional` has country codes with `twitter_rank` / `x_rank`
- [ ] `tranco.twitter` and `tranco.x` are arrays
- [ ] `trends` is `null` (deferred to Phase 9)
- [ ] `updated_at` is a timestamp
- [ ] No `errors` array (if all sources succeeded)

### Step 4: Test KV caching

```bash
# First request should fetch live (may take 1-3 seconds)
time curl -s http://localhost:8788/api/data > /dev/null

# Second request should hit KV cache (~instant)
time curl -s http://localhost:8788/api/data > /dev/null

# Test cache bust — should bypass cache and fetch live again
time curl -s "http://localhost:8788/api/data?refresh=true" > /dev/null
```

**What to check:**
- [ ] First request is slower (live API calls)
- [ ] Second request is near-instant (KV cache hit)
- [ ] `?refresh=true` forces a fresh fetch (slower again)

## Quick Reference — Local Commands

| Command | What it does |
|---|---|
| `npm run dev` | Start Vite-only dev server (http://localhost:5173, hot reload, mock data) |
| `npm run dev:full` | Build + start Wrangler Pages dev server (http://localhost:8788, real API + KV) |
| `npm run build` | Build production bundle to `dist/` |
| `npm run preview` | Preview built bundle with Wrangler locally (same as dev:full but no rebuild) |
| `npm run lint` | Run ESLint |
| `npm run deploy` | Deploy to Cloudflare Pages (Phase 6) |
| `curl http://localhost:8788/api/data` | Test API endpoint (requires dev:full or preview) |
| `curl "http://localhost:8788/api/data?refresh=true"` | Force cache refresh |
| `rm -rf .wrangler/state` | Reset local KV to empty |

## Troubleshooting

**"KV namespace not found" or binding error:**
The `wrangler.toml` has `id = "<your-kv-namespace-id>"`. For local dev, Wrangler creates a local KV store automatically — this placeholder ID works fine locally. You only need a real ID for production deploy (Phase 6).

**API returns `radar: null` with auth error:**
Your `CLOUDFLARE_RADAR_TOKEN` in `.dev.vars` is missing or invalid. Get a new token from https://dash.cloudflare.com/profile/api-tokens.

**Charts don't render / blank page:**
Check browser console for errors. If the API fails and mock data fallback also fails, the error state should display. Check that `src/utils/mockData.json` exists.

**Port 8788 already in use:**
Kill the existing process: `lsof -ti:8788 | xargs kill` then retry.

**Dark mode not toggling:**
Check that `<html>` element has the `dark` class toggling in DevTools → Elements. The `@variant dark` in `index.css` uses `:where(.dark, .dark *)` selector.
