## Overview

Deploy the app to Cloudflare Pages with Workers KV caching, production secrets, and custom domain `didtwitterdie.com`.

---

## Before You Start (manual steps required)

These must be done **before** running any commands. They require a browser and your Cloudflare account.

### 1. Create a Cloudflare account
If you don't have one: https://dash.cloudflare.com/sign-up (free tier is sufficient)

### 2. Generate a Cloudflare API token for Radar
1. Go to https://dash.cloudflare.com/profile/api-tokens
2. Click **Create Token** → **Custom token**
3. Set permission: `Account` → `Account Analytics` → `Read`
4. Click **Continue to summary** → **Create Token**
5. **Copy the token now** — you won't see it again
6. Save it somewhere safe, you'll need it in Step 3 and Step 5 below

---

## Step 1 — Install and authenticate Wrangler

```bash
npx wrangler login
```

This opens a browser window to authorize Wrangler with your Cloudflare account.

> **STOP**: Wait for browser auth to complete before continuing.

Verify it worked:
```bash
npx wrangler whoami
```

---

## Step 2 — Create the KV namespace

```bash
npx wrangler kv namespace create CACHE
```

Output will look like:
```
✅ Created namespace "didtwitterdie-CACHE" with ID "abc123..."

Add this to your wrangler.toml:
[[kv_namespaces]]
binding = "CACHE"
id = "abc123..."
```

> **STOP**: Copy the `id` value and update `wrangler.toml`:
> Replace `id = "<your-kv-namespace-id>"` with the real ID.
> Commit this change — the KV namespace ID is not a secret.

---

## Step 3 — Set the Radar API token secret (production)

```bash
npx wrangler pages secret put CLOUDFLARE_RADAR_TOKEN
```

Wrangler will prompt you to paste the token value (from Before You Start → Step 2).

> **STOP**: Paste the token when prompted. This stores it securely in the Cloudflare dashboard — it never touches your local files.

---

## Step 4 — Build and deploy

```bash
npm run build
npm run deploy
```

The first deploy creates the Pages project automatically. Note the deployment URL (e.g. `https://didtwitterdie.pages.dev`).

---

## Step 5 — Connect GitHub repo for auto-deploys (optional but recommended)

1. Go to https://dash.cloudflare.com → **Workers & Pages**
2. Find the `didtwitterdie` project → **Settings** → **Git integration**
3. Connect to your GitHub repo `egekocabas/didtwitterdie`
4. Build settings:
   - Build command: `npm run build`
   - Build output directory: `dist`
5. Save — every push to `main` will now auto-deploy

> **STOP**: After connecting, verify the Radar token secret is set under **Settings → Environment variables** (it should be there from Step 3, but double-check).

---

## Step 6 — Add custom domain

1. Go to the Pages project → **Custom domains** → **Set up a custom domain**
2. Enter `didtwitterdie.com`
3. Follow the DNS instructions (add a CNAME record at your domain registrar)
4. Wait for DNS propagation (can take up to 24h, usually faster)

> **STOP**: DNS propagation must complete before the custom domain works. The `.pages.dev` URL works immediately.

## Step 7 — Verify everything works

```bash
# Check the live API response
curl https://didtwitterdie.pages.dev/api/data | jq .
```

Verify:
- `radar.twitter.bucket` and `radar.x.bucket` are not null (Radar API token works)
- `tranco.twitter` and `tranco.x` arrays have data (Tranco API + seed data works)
- `updated_at` is a recent timestamp

Also verify in browser:
- [ ] Site loads at `https://didtwitterdie.pages.dev`
- [ ] Dark mode toggle works
- [ ] All 4 sections render (Hero, Ranking chart, Google Trends embed, Verdict)
- [ ] Hero shows real rank numbers (not mock data)
- [ ] Footer shows "Last updated" timestamp

---

## Step 8 — Verify KV caching

```bash
# Read the cached data from production KV
npx wrangler kv key get --binding=CACHE --remote "all_data"
```

Should return the full JSON response. If empty, hit the API endpoint once to trigger a cache-miss fill, then try again.
