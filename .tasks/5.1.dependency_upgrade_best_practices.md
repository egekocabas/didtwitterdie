## Steps

### Part A: Dependency Upgrades & Pinned Versions

#### 1. Pin all versions in `package.json`

Remove `^` prefix from every dependency and devDependency. Use exact versions for reproducible builds (e.g., `"react": "19.2.4"` not `"^19.2.4"`).

#### 2. Do NOT upgrade ESLint to v10

**Reason:** `eslint-plugin-react-hooks@7.0.1` (latest stable) declares `peerDependencies: eslint ^3-^9` only. It does not support ESLint 10. There is no stable release with ESLint 10 support yet (only canary builds). Upgrading would require `--force` or `--legacy-peer-deps`.

**Action:** Stay on `eslint@9.39.4` and `@eslint/js@9.39.4`. Pin at exact versions. Revisit when `eslint-plugin-react-hooks` releases a stable version with ESLint 10 support.

#### 3. Upgrade wrangler 4.77.0 → 4.78.0

- `npm install -D wrangler@4.78.0 --save-exact`

#### 4. Upgrade vite 8.0.1 → 8.0.3

- `npm install -D vite@8.0.3 --save-exact`

#### 5. Run `npm install` to regenerate lock file

### Part B: ESLint & Editor Config Best Practices

#### 6. Update `ecmaVersion` in `eslint.config.js`

Change `ecmaVersion: 2020` → `ecmaVersion: "latest"`. The project uses ES2022+ features (optional chaining, nullish coalescing) and Vite targets modern browsers.

#### 7. Create `jsconfig.json`

Add editor IntelliSense and `@/` path alias:
```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "baseUrl": ".",
    "paths": { "@/*": ["src/*"] }
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```

#### 8. Update `vite.config.js` — add path alias

```js
resolve: {
  alias: { "@": "/src" }
}
```

### Part C: Tailwind / Dark Mode Fixes

#### 9. Add CSS variables for chart theming in `src/index.css`

Recharts uses inline styles, not Tailwind classes. Define CSS variables that adapt to dark mode:

```css
:root {
  --chart-grid: #e5e7eb;
  --chart-x-stroke: #111827;
  --tooltip-bg: #ffffff;
  --tooltip-border: #e5e7eb;
}
.dark {
  --chart-grid: #374151;
  --chart-x-stroke: #9ca3af;
  --tooltip-bg: #1f2937;
  --tooltip-border: #374151;
}
```

#### 10. Fix `DnsChart.jsx` — CRITICAL dark mode issues

- **X.com line** `stroke="#111827"` → `stroke="var(--chart-x-stroke)"` — invisible on dark bg
- **CartesianGrid** `stroke="#e5e7eb"` → `stroke="var(--chart-grid)"`
- **Tooltip** `backgroundColor` → `var(--tooltip-bg)`, border → `var(--tooltip-border)`

#### 11. Fix `RankingChart.jsx` — same issues as DnsChart

Same three fixes: X.com line stroke, grid stroke, tooltip styling.

#### 12. Fix `RegionalChart.jsx` — same issues

- **X.com bar** `fill="#111827"` → `fill="var(--chart-x-stroke)"`
- **CartesianGrid** and **Tooltip** — same CSS variable fixes

#### 13. Fix `TimeRangeSelector.jsx` — minor

Add `dark:text-gray-400` to inactive button state (currently only `text-gray-500`).

#### 14. Fix `App.jsx` ErrorState — minor

Add `dark:text-red-400` to the `text-red-500` error message.
