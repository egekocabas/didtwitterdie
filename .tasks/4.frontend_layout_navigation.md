## Goal

Build the frontend shell: page wrapper with dark mode toggle, data-fetching hook, loading skeletons, single-page scroll structure in App.jsx, and footer. After this phase, the app loads data (mock or live), shows loading states, and renders placeholder sections ready for Phase 5 chart components.

## Steps

### 1. Create `src/hooks/useData.js` — data fetching hook

Custom hook that fetches `/api/data` on mount with loading/error states. Falls back to mock data in development when the API is unavailable.

```jsx
export function useData() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch("/api/data")
      .then(res => {
        if (!res.ok) throw new Error(`API error: ${res.status}`);
        return res.json();
      })
      .then(d => setData(d))
      .catch(err => setError(err.message))
      .finally(() => setLoading(false));
  }, []);

  return { data, loading, error };
}
```

- Returns `{ data, loading, error }`
- Components consume this hook via props from App.jsx (single fetch, not per-component)

Wraps the entire page. Responsibilities:

- Font and background classes already set up in `index.css` (Inter, system-ui)

Dark mode logic:

Visual: small toggle button in top-right corner (sun/moon icon, no emoji — use text or SVG).

### 3. Update `src/App.jsx` — single-page scroll structure

Replace the current placeholder with the full page structure:

```jsx
function App() {
  const { data, loading, error } = useData();

  if (loading) return <Layout><LoadingSkeleton /></Layout>;
  if (error) return <Layout><ErrorState message={error} /></Layout>;

  return (
    <Layout>
      <HeroSection ... />
      <DnsChart ... />
      <TrendsPlaceholder />
      <RankingChart ... />
      <RegionalChart ... />
      <VerdictSection ... />
      <Footer updatedAt={data.updated_at} />
    </Layout>
  );
}
```

For Phase 4, chart components will be simple placeholder divs with section titles (actual charts come in Phase 5). Each section receives its slice of `data` as props.

### 4. Create `src/components/LoadingSkeleton.jsx` — skeleton screens

Animated skeleton UI shown while data loads. Mimics the final page layout:
- Pulsing gray bars for the hero numbers
- Pulsing rectangles for chart areas
- Uses Tailwind `animate-pulse` on gray blocks

Should feel like the real page is about to appear (not a spinner).

### 5. Create `src/components/Footer.jsx` — credits and timestamp

Bottom of the page with:
- Data source credits: "Data from Cloudflare Radar and Tranco List"
- "Last updated" timestamp formatted from `data.updated_at`
- CC BY-NC 4.0 attribution for Cloudflare Radar data (required by license)
- Link to the GitHub repo (optional)

Styled: muted text, small font, subtle top border or spacing.
